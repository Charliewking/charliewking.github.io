Part 1 - Accessing your webspace in Stdlinux

                Basic Steps –

1.       Log in to Stdlinux

2.       Create index.html

3.       Change files so they can be accessed from the web

 

Step 1 – Log in to Stdlinux

 

There are a few ways to do this.  If you are on a lab computer you can get in through the stdlinux desktop environment by clicking the icon.

 

You could also log in using an ssh client or a terminal window in any unix environment using the command

 

ssh stdlinux.cse.ohio-state.edu

 

After that you will be prompted for your username, this is your cse username so probably parts of your last name and maybe first name, mine is kingcha, not king.1380. 

 

MAKE SURE YOU REMEMBER THIS USERNAME!

 

You will need it later, it will also be the way you access your webspace.  If you want to you can try it out now and see what your web page currently looks like.  Here is the address:

 

http://www.cse.ohio-state.edu/~username/

 

Just replace ‘username’ with your username and you’re there.  Most likely you will some kind of cse page not found message with a  picture of Dreese hall on it.  That’s okay, it just means you don’t have a place for people to go yet, but you’re about to!

 

Step 2 – Create index.html

 

So now that you have accessed your stdlinux somehow, you can look at your file system.  Open a terminal window and type ls and hit enter.  What you should see now is something like this:

 

 

The only thing that matters is that WWW directory for our purposes.  If you don’t see a WWW in that blue color, type:

 

mkdir WWW

 

Now type:

 

cd WWW

 

You are in your webpage directory!  This is the folder the server pulls from when it someone requests your webpage so you need to put some stuff in here.  You can do that by creating the basic page in here.  Type:

 

gedit index.html &

 

now the gedit editor app should come up so you can start writing.  You are well on your way, this blank canvas will soon become your personal web page. 

 

Step 3 – Change files so they be accessed from the web

 

The last thing you have to do before you start creating is change the access level of your files.  You could choose to do this last, but you may as well learn it here, you will have to do it every time you add a new file to the WWW directory.  Here is how it goes, use the chmod command to change the access rights of your files so people can see them from their web browser.  First type:

 

Chmod a+rx index.html

 

Also you will have to go up the home directory and do the same thing to your WWW directory.  Type:

 

cd

 

Now you are in the home directory so type:

 

Chmod a+rx WWW

 

Now you can go back into the WWW directory if you want.  By the way chmod a+rx means “change mode” that doesn’t really help so you can think of it as “change permissions” and the a+rx means ‘a’ means ‘all’, as in all users.  ‘+’ means ‘add permissions’.  ‘rx’ means ‘reading and execution’.  So basically, you have given other users the ability to read and access your files.   Just FYI.

Step 1 – Filling in your index.html document

 

Now you get to actually start designing your page.  This is the time to start thinking about what you want your page to look like.  Maybe go to a site you like and try to copy that a little bit.  Think about the colors you might want to use.  There are a lot of places to go to find out what the html markup for colors is, just search for it and you will find what you need. 

 

The first step here is to declare to the browsers that this is an html5 page.  To do this type:

 

<!DOCTYPE html>

 

Now you can set up the basics of a page.  Every page has an Html element, a Head element, and a Body element.  A barebones Html web page looks like this:

 

<!DOCTYPE html>

<HTML lang="en-US">

<HEAD>
    <!-- write some code here.  This is a comment in HTML by the way -->

     <p> This is some text in the header </p>
</HEAD>
<BODY>
     <!-- write some code here in the body. -->
     <p> Hello World </p>
</BODY>
</HTML>

 

There you have it.  Paste that in to your index.html file, hit save and go to your page.  Here is what it should appear as in your browser:

 

 

Now you’re done!  I’m just kidding, you probably want to add some stuff first.

 

Tags

While we are here, the <HEAD> format is called a ‘tag’.  In HTML, markup tags are keywords. Tags come in pairs and are surrounded by angle brackets. Tag pairs consist of a start tag like <p> and an end tag like </p>. Notice that the end tag has a forward slash before the tag name.

 

Other useful tags are:

 

<title> tag defines the title of the web page</title>

 

The text inside the <title> tags will appear on the tab bar in your web browser.

<h1> to <h6> tags define different headings, where lower headings mean higher heading importance, <h1> being the most important header or the biggest.

 

<p> tags define paragraphs</p>

 

<strong> tags flag their content as “important”, usually rendering this text in bold, but this behavior can be changed by manipulating the style of the element, more on this a little later.

 

<em> tags also flag their contents as “important” but their rendering here is in italics, also flexible to change. </em>

 

<div> tags are useful for grouping information in a document for formatting purposes, they can have their own elements and their own HTML id.  We will go over how to use <div> tags to organize your page in sections 4 and 5.

 

Elements

An HTML element consists of everything between the start tag and the end tag, tags included. The following is an example of an entire HTML element:

 

<h1> This is a header element. </h1>

 

With those two simple things in mind let’s make a web page!

 

Step 2 – Putting some personal information on your page

 

First we will put your name in the page as the header.  To do this, start with the old html basic layout and add your name as the header.

 

<!DOCTYPE html>

<HTML lang="en-US">

<HEAD>
     <h1> YOUR NAME HERE </h1>
</HEAD>
<BODY>
     <!-- write some code here in the body. -->
     <p> Hello World </p>
</BODY>
</HTML>

 

At this point you can save your site and navigate to your page just to make sure something has happened.  Remember that URL is:

 

http://www.cse.ohio-state.edu/~username/

 

Here is what you should see:

 

 

 

Once you verify that your site actually exists, move on.  If you got a message saying your page doesn’t exist or you don’t have access to it try going back through Part 1: Step 3.

 

Next add a little more detail about yourself.  We will do this using a <ul> or ‘unordered list’.  Yes, there is also an ‘ordered list’ which is <ol>, but that lists things with numbering, and we just want bullet points, so we will stick with <ul> for now.  The <ul> </ul> tags just set up a list, then you can fill it with <li> or ‘list item’ elements with text.  It will all look something like this:

 

<ul>

     <li> I am a Computer Science student at THE Ohio State University</li>

     <li> I like watching football and playing tennis </li>

     <li> One day I would like to be a computer programmer </li>

</ul>

 

Place that block between your <BODY></BODY> tags, save and reload your page and you should see your list show up. 

 

 

 

Now you have a very basic resume-like website.  From here you can do all kinds of things.  We will take you through a few more additions before letting you go out on your own.

 

Step 3 – Displaying a picture of yourself

 

Find the best picture of yourself.  It can be a facebook picture or just google yourself and pick the first one that looks decent, download it and save it in your WWW directory.  If you are using an internet browser on the windows environment you can save it to:

 

Z:\WWW

 

Or you can use an ftp client to throw the picture into your WWW folder.  Also don’t forget to change the permissions on the image with the command:

 

chmod a+rx ImageName.ImageExtension

 

If you don’t know what the name of the image is you can always do an ‘ls’ command and it should show up in the console printout.

 

Now we create an <img> tag on your page.  As you probably guessed, ‘img’ stands for ‘image’, so type something like this inside your <BODY> element:

 

<img src="Charlie.png” alt=”Picture of Me”></img>

 

Now as long as your image is set with the proper permissions, you should see your beaming face on your site.  The purpose of the alt=”Picture of Me” part is to insure that when your picture doesn’t load, something other than the name of the image will be displayed to let site visitors know what is supposed to be there.  Some of you may actually get to experience this, if your picture didn’t load it’s almost certainly because you didn’t change the permissions properly.  Try it again and your picture should show up!

 

 

Part 3 - Hosting documents on your page

 

                Basic Steps –

1.       Place your resume or some other document in your WWW folder

2.       Make it so everyone can see your document

3.       Place a link to your document on your page

 

Step 1 – Place your resume or some other document in your WWW folder

 

Similar to what you did for your image, go find a .doc or .pdf of your resume and save it in your WWW directory, this step is pretty easy. 

 

If you’re having trouble, you can access your WWW directory if you’re using a lab computer by going to My Computer -> Home(Z:) -> WWW and save it in there.

 

Step 2 – Make it so everyone can see your document

 

Again, just like with the image from the previous part, use the chmod command to change the permissions for the document you want to display.  Something like this:

 

chmod a+rx FileName.FileExtension

 

Once you’ve done that, it now exists to be viewed from the web.  You can try it by going to:

 

http://www.cse.ohio-state.edu/~username/FileName.FileExtension

 

Now you just want to make it a little easier for a visitor to find this document, because they probably don’t know you named your resume “JoeisthebestEVER.doc”.  No judgments here.

 

Step 3 – Place a link to your document on your page

 

To make a link to your resume you use an ‘a’ element with a hyper-reference or ‘href’. 

 

Side note: ‘hyper’ is used a lot in web stuff. HTML actually stands for Hyper Text Markup Language.

 

It will look something like this:

 

<a id="resume" href=" JoeisthebestEVER.doc " >Here is my current Resume </a>

 

A few things happened here.  First is the id modifier.  Ids help keep your page in order, and let you know at a quick glance what you’re looking at if things start to get cluttered.  Also in HTML5 you can reference an id to make it so site visitors can click something to take them straight to that id tag.  It’s not necessary for creating a website to include ids, it’s just best practice. 

 

The next thing is the href.  As I said before it’s a hyper reference, so it will actually take the visitor to another page entirely when they click it.  In this case it just takes them to the document itself and the browser will display it in a manner it sees fit.  For documents and PDFs it will often take up the entire browser window. 

 

Then the text in the element and the </a> article element close tag.  The text is what will actually be displayed in the hyperlink when it’s marked up on your page.  It will look like this:

 

Here is my current Resume

 

Hrefs are also used for taking visitors to other sites.  Down the road if you want to make another page on your web space you would create something like “Page2.html”, save it in your WWW directory, change the permissions, and create a link on your front page that looks like this:

 

<a id="Page2" href=" Page2.html " > Page 2 </a>

 

That is a very basic primer to making an html webpage on the CSE webspace.  From this point on we will just be dealing with HTML/CSS styling.  If you are okay with having a very basic page, you can stop reading now.  If you want to make your page a little more nice-looking and learn about CSS, keep reading!
